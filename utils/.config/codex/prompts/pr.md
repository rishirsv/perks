Create a branch named `<project_name>/<description>` and create a pull request for the changes you made.
If you are already on a siloed feature branch or worktree, you can skip the step of creating a new branch and just create a pull request for the changes you made.

Example: 
If you were working on a feature called `add hero animations` in the project `capital` you would create a branch named `capital/add-hero-animations` and create a pull request for the changes you made.

If you were working on a feature called `add hero animations` in the project `capital` and you were already on the branch `capital/add-hero-animations`, you can skip the step of creating a new branch and just create a pull request for the changes you made.

Commit the changes you made to the branch and create a pull request for the changes you made.
Use a clear and concise commit message that is easy to recall and understand.
