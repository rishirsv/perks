# Competitor teardown template

Use this only when the user explicitly requests competitor research.

## Capture set
For each competitor, collect screenshots for:
- Key pages/flows relevant to the user goal
- Key components that embody their system (tables, cards, nav, forms)
- Key states (hover/focus/empty/loading/error)

Keep viewports consistent so comparisons are meaningful.

## Pattern analysis
- Layout system: grid, density, spacing rhythm, section transitions
- Typography: number of levels, emphasis strategy, scan anchors
- Color/tokens: dominant vs accent use, semantic colors, consistency
- Motion: where it exists and what job it does
- Content strategy: headline/label style, helper text, empty/error tone

## Recommend a template/system to adapt
Extract principles, not pixels:
- “Copy” structure and decisions (hierarchy, tokens, rhythm)
- Do not copy exact layouts, illustrations, brand assets, or distinctive visuals

## Implementation plan
List a short plan that fits the current codebase:
- start with tokens and spacing scale
- then update 1–2 core components
- then expand across the UI with screenshot validation each step
