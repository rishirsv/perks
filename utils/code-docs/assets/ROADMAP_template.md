## How to write good feature descriptions

Aim for three to four bullets. Keep it concrete and outcome-oriented.

Good:
- “Add a billing screen that shows current plan, next renewal date, and lets users change plans.”

Vague:
- “Improve billing UX.”

Avoid:

* Implementation details or technical design.
* Task lists or step-by-step plans.
* Vague phrases like “improves UX” without explaining what changes.

A good description should let someone understand *why the feature exists* and *what changes once it ships*, without knowing how it will be built.
