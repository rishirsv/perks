# Task Fit Rubric

Use autoresearch only when the task earns it.

## Good fit

- The task has a concrete artifact or output.
- The task has a real verification path.
- The work is likely to benefit from multiple focused tries.
- The cost of a failed iteration is low enough to learn from.

## Weak fit

- The task is mostly exploratory.
- The task needs open-ended synthesis more than iteration.
- The task depends on user taste with no rubric.

## Poor fit

- No honest verify path exists.
- The outcome is pure brainstorming.
- The problem is really a research brief or a formal plan request.

## Decision

- If the task is a good fit, continue into setup.
- If the task is a weak fit, consider a single bounded pass with explicit caveats.
- If the task is a poor fit, route to a better skill.
