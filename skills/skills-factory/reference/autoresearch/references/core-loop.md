# Core Loop

Use the same five-part loop every time:

1. shape the task
2. establish the baseline
3. make one focused change
4. verify and guard
5. keep or discard and log

## Core principles

- Constrain the scope before touching files.
- Define success in a way the agent can actually measure.
- Read recent git history when git exists.
- Keep one hypothesis per iteration.
- Record lessons from losing runs instead of silently forgetting them.

## Default loop order

```text
setup -> baseline -> iterate -> verify -> guard -> keep/discard -> summarize
```

Use this order unless the task has a strong reason to differ.
