# Docs Index

This folder was reorganized for faster navigation and cleaner knowledge management.

## Structure

- `product/`
  - `spec-spa-assistant.md` — product requirements, UX flow, playbook expectations, and acceptance criteria.
- `planning/`
  - `plan-spa-assistant-v2.md` — implementation plan and milestone breakdown.
- `playbooks/`
  - `spa-playbooks.md` — canonical playbook definitions used for extraction workflows.
- `knowledge/`
  - `market-benchmarks.md` — market benchmark references for pricing, protection, and negotiation analysis.
- `research/`
  - `research-*.md` and `research-spa-example.md` — research notes and topic prompts used by the playbooks.
- `patches/`
  - `.diff` records used during historical design iterations.
- `engineering/`
  - `definitions-extractor/plan-definitions-extractor.md` — extractor implementation plan.

## Notes

- The `docs/` folder intentionally avoids long flat lists of Markdown assets and groups content by purpose.
- If links refer to historical paths, use `../` from the current nested location (for example from `docs/planning/` or `docs/product/`).
