# Autopilot Batch Brief: Remaining Reports

## Objective
- Extract verbatim report text for every remaining source report.
- Perform full-report cleanup on every report before handoff.
- Enforce fail-closed QA gates before marking pass.

## Non-Negotiable Quality Rules
- Cleanup is mandatory for the whole report, not one section.
- No summarization or paraphrasing; keep source wording.
- Remove engagement-letter/legal/cover/glossary/navigation boilerplate from content sections.
- Use source-text artifacts and montage to anchor section placement.
- Required gates before pass: provenance_exact_match, markdown_trace_sync, section_completeness, executive_summary_cleanup, cleanup_quality.
- Never mark a story `passes=true` while extracted markdown still has `EXTRACTION_STATUS: needs_revision`.
- A story is `pass` only after manual montage reconciliation is complete and recorded in review notes with page-level references.
- If section placement is uncertain, keep exact wording and move lines to the closest canonical section; do not invent connective summary text.
- Append one extraction learning entry to `AGENTS.md` per report.

## Single-Report Execution Recipe (repeat per story)
1. Create isolated report staging dir with one source file.
2. Run pipeline in strict mode against that isolated dir.
3. Generate source text artifacts (`scripts/extract_source_text.py --verbatim-mode strict`).
4. Do full-report manual cleanup and template alignment in `extracted/<report-id>.md`.
5. Run provenance and gates QA.
6. Perform montage-to-markdown reconciliation and log concrete page IDs in review notes.
7. Complete review notes checklist.
8. Mark reviewed status (`pass` only when gates pass).
9. Append AGENTS learning entry for this report.

## Commands Template
```bash
report_file="<FILENAME>"
report_id="<REPORT_ID>"
stage_dir="extracted/tmp/autopilot-${report_id}/reports"
mkdir -p "$stage_dir"
cp "reports/${report_file}" "$stage_dir/"

./.venv/bin/python -m ts_report_writer.pipeline run \
  --reports-dir "$stage_dir" \
  --extracted-dir extracted \
  --verification-dir extracted/verification \
  --manifests-dir extracted/manifests \
  --tmp-dir extracted/tmp \
  --max-reports 1 \
  --verbatim-mode strict \
  --evidence-profile full \
  --fail-closed

./.venv/bin/python scripts/extract_source_text.py \
  --source "reports/${report_file}" \
  --out-dir "extracted/verification/${report_id}/source-text" \
  --verbatim-mode strict

./.venv/bin/python scripts/qa_provenance.py \
  --markdown "extracted/${report_id}.md" \
  --source-manifest "extracted/verification/${report_id}/source-text/manifest.json" \
  --out-dir "extracted/verification/${report_id}/qa"

./.venv/bin/python scripts/qa_gates.py --report-id "$report_id"
```
