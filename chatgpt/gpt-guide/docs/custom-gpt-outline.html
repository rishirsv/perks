The Mindset Shift

02 / 14

Literal Instruction-Following = Be Explicit

"Existing prompts optimized for other models may not immediately work... existing instructions are followed more closely and implicit rules are no longer being as strongly inferred." — OpenAI GPT-4.1 Prompting Guide

What This Means

Before Now
Model infers unstated intentions Model follows instructions literally
Vague prompts "work" (sometimes) Vague prompts produce unexpected results
Rules-heavy, hope model "gets it" Workflows must be explicit

Practical Implication

❌ Vague
"Be helpful and professional" "Summarize this document"
✓ Explicit
"Respond in 2-4 sentences. Use active voice. No hedging language." "Summarize in 5 bullet points. Each bullet: one key finding + one implication."

Define Constraints Before Building

Why write a spec first: Forces clarity on scope, constraints, edge cases. Creates testable success criteria. Prevents scope creep. Documents decisions for future maintainers.

Spec Template

# [GPT Name] — Specification ## Purpose One sentence: What does this GPT do? ## Type ☐ Deterministic (same output every time) ☐ Workflow (multi-step, adaptive) ## Inputs What does the user provide? ## Outputs What format/schema? Expected length? ## Constraints (NON-NEGOTIABLE) What must it NEVER do? What must it ALWAYS do? ## Workflow Step-by-step process ## Validation How do we know output is correct?

Real Example — Meeting Intelligence

## Purpose Produce comprehensive meeting notes from transcripts. ## Type ☑ Workflow (adapts to meeting type) ## Constraints (NON-NEGOTIABLE) - Zero inference: every statement grounded - Never fabricate numbers or claims - No speaker attribution in notes - Preserve exact quotes for critical statements ## Workflow 1. Classify meeting type (FDD vs General) 2. Read transcript, build topic outline 3. Compose notes using template 4. Validate against checklist ## Validation - Every major topic appears somewhere - All material figures captured - No fabricated numbers

Guardrails Deep Dive

06 / 14

Tell It What NOT To Do, AND What TO Do Instead

TS Copywriter <Must_Avoid>

1. Assurance Terms: "ensure," "accurate," "fair," "reasonable" 2. Vague Magnitude: "significant," "substantial," "material" 3. Purpose Statements: "in order to," "designed to" 4. Process Descriptions: "we analyzed," "we reviewed" 5. Opinion Language: "we believe," "appears to be"

TS Copywriter <Word_Substitutions>

"ensure/ensures" → "results in," "creates" "significant" → [exact percentage or amount] "we analyzed" → "the data shows" "we believe" → "the data indicates" "it was noted" → "[subject] showed/indicated"

Meeting Intelligence Prohibited Patterns

❌ Dialogue Reconstruction
"When asked about X, the response was Y" "X was discussed; they said Y" "Question raised about X; no answer received"
✓ Direct Statement
"Y." "Y." "X remains unconfirmed (open item)."
Key Insight: Ban lists alone aren't enough. Provide substitutions — tell the model what TO say, not just what NOT to say.

Validation Gates

07 / 14

Quality Isn't Vibes — Force the Model to Check Its Work

TS Copywriter 8-Point Protocol

1. Every claim anchored to a number/source 2. No passive voice without a clear subject 3. No assurance or opinion language 4. Consistent use of standard terms (NWC, COGS, EBITDA) 5. No restricted words from Must_Avoid 6. Apply all Word_Substitutions 7. No qualitative judgments 8. Paragraph structure per Style_Guide

Meeting Intelligence Flight Check

Before finalizing, check: ☐ Appropriate template used verbatim ☐ Every major topic in transcript appears somewhere ☐ All material figures/policies/risks are captured ☐ No fabricated numbers or unstated assumptions ☐ For FDD: Key Takeaways appears before Meeting Notes in each topic

Controlling Verbosity & Disclosure

08 / 14

Output Size + Workflow Depth — One Coherent Story

Control What It Does Example GPT
Verbosity Match output length to input detail KDN Tasks
Prompt Brevity Keep prompts short, behavior-changing only MetaPrompt
Progressive Disclosure Reveal complexity only when needed FDD Researcher

Verbosity Control (KDN Tasks)

Set verbosity to LOW. Be terse and direct. - 2-4 numbered tasks per email, 2-4 substeps MAXIMUM - Each substep: ONE short sentence - Mirror the user's specificity level
❌ Over-elaborates brief input
User: "Roll forward the databook through July" 1. Jul-25 databook roll-forward: a. Add Jul-25 column to all IS tabs b. Add Jul-25 column to all BS tabs c. Add TTM-Jul 25 column d. Verify all formulas extend correctly
✓ Match brevity
User: "Roll forward the databook through July" 1. Jul-25 databook roll-forward: a. Add Jul-25 column to all IS and BS tabs; add TTM Jul-25 totals

Template-Driven Architecture

Structure Enables Consistency Without Rigidity

MetaPrompt 3-Template Routing

Essential: 80-150 words — single compact artifact Standard: 120-220 words — multi-section documents Research: 180-350 words — deep research with citations Routing: - Default to Essential unless output has named sections/groups - When unsure, bias toward the simpler - Never mention templates or routing in final prompt

FDD Meeting Template Structure

Mode Detection & Routing

10 / 14

One GPT, Multiple Jobs — No Menus

TS Copywriter Keyword Routing

Default mode is Rewrite unless the user input contains "draft" (case-insensitive).

KDN Tasks Request Type Detection

Determine request type: - Initiation: new deal, no prior KDN nights, or explicit "new project" - Nightly: ongoing deal; nightly instructions - Restructure: turn rough notes into structured instructions - Polish: refine a draft email - Document analysis: user asks for task recommendations

Meeting Intelligence Type Selection

Classify as: - FDD: focus on financial statements, EBITDA, revenue, deal diligence - General Business: all other meetings # Key Insight: Detect intent from content. Only ask user if truly ambiguous.

✓ Specific
"Update *Project Academy Databook_v5.xlsx* tab 'IS|Combined'"

{Falcon_Databook_v2.xlsx}

"the databook"

"Differences were adjusted by the Company."
✓ Active voice
"The Company adjusted differences of $2.3M."
❌ Vague magnitude
"Working capital went down."
