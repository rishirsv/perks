# AGENTS Maintenance

## Purpose

Use this reference when creating or revising `AGENTS.md` so it stays concise, current, and worth reading.

## Core rules

- Keep `AGENTS.md` short.
- Prefer about 80-120 lines.
- Treat 150 lines as a hard cap.
- If you add an important rule, remove or compress something else.

## What belongs in `AGENTS.md`

- the operating contract for contributors and agents
- the repo map for the most important docs and commands
- critical workflow rules
- safety boundaries
- verification loop

## What does not belong

- changelog-style history
- duplicated policy that already lives in deeper docs
- speculative or unenforced rules
- long narrative explanations
- placeholders for files that do not exist

## Update discipline

Every time you edit `AGENTS.md`, also check whether you need to:

- remove stale rules
- update commands
- update paths and links
- update doc names
- remove duplicate or conflicting guidance
- tighten wording

## Preferred editing pattern

1. Read the whole file, not only the section you plan to change.
2. Identify stale, duplicated, or weak rules first.
3. Replace or compress before appending.
4. Keep each rule concrete and imperative.
5. Link out to deeper docs instead of embedding long detail.

## Good outcomes

`AGENTS.md` should feel:

- short enough to read in one pass
- current enough to trust
- specific enough to act on
- stable enough not to need daily churn
