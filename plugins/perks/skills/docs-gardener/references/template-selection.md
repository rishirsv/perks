## Selection rules

- If the canonical doc already exists, preserve its structure and tone unless the task is explicitly to re-template it.
- Use templates to create new durable docs or when the existing file is clearly scaffold-like and needs replacement.
- Do not reintroduce issue-specific or tech-debt-specific templates when the repo is moving to a unified work tracker.
