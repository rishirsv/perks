# WORK-TRACKER

## Purpose

Single operational backlog for actionable repo problems.

Use this file for:

- bugs
- tech debt
- docs debt
- contract gaps
- reliability gaps
- follow-up cleanup

## Intake Rules

- Add every actionable problem here instead of creating a separate issue or debt registry.
- Use one entry schema for all work.
- Link accepted execution work to an exec plan and keep the tracker summary short.
- Keep resolved items concise and recent; do not let this file become a second wiki.

## Priority Rubric

- `P0`: critical production blocker, data loss, or security-critical risk
- `P1`: major user-facing breakage or release-gating regression
- `P2`: meaningful defect, docs gap, or maintenance issue with clear impact
- `P3`: minor defect, polish, or cleanup

## Status Values

- `open`
- `triaged`
- `in-progress`
- `blocked`
- `resolved`
