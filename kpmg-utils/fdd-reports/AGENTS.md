## Learned User Preferences
- Prefer one adaptive layout per section contract, with scaling rules, instead of multiple named layout variants.
- Merge layout-to-slot recipe guidance into the Layout (or slot-shapes) section to keep contracts concise.
- In net debt modules, use `Other considerations` for potentially debt-like items that are not fully quantifiable; it may include open items but should not read as a generic open-items list.
- Do not include a comments column in net debt schedules; explanatory treatment belongs in `Debt-like adjustments and rationale`.
- Write net debt adjustments as numbered, full-sentence entries using `Item ($x): Relates to ...` with explicit treatment and closing implications.
- Keep render skeletons fully copy-ready with complete placeholder sentences, and keep full examples as fully worked examples with filled values.
